🔹 AIM
To perform image acquisition and analysis using FTK Imager and Autopsy

🔹 THEORY (5–6 lines, simple)
Disk imaging means creating an exact copy of a storage device.
It is used in digital forensics to preserve evidence.
Image formats like E01 ensure data integrity and compression.
FTK Imager is used for acquisition (creating image).
Autopsy is used for analysis (view files, deleted data, activity).
Image should be stored on a different drive to avoid data overwrite.

🔹 PROCEDURE (SHORT + PRACTICAL)
🧪 PART 1: IMAGE ACQUISITION (FTK Imager)
Open FTK Imager → Run as Administrator
Click File → Create Disk Image
Select:
👉 Logical Drive (C:)
Click Next
Select E01 format → Next
Fill basic details → Next
Click Add → Choose location:
❗ Select USB / D: drive (NOT C:)
Click Finish → Start
📸 Screenshot: Image creation + completion

🔍 PART 2: VERIFY IMAGE (FTK Imager)
File → Add Evidence Item
Select Image File
Open .E01 file
Expand folders and view files
📸 Screenshot: Files visible

🔎 PART 3: ANALYSIS (Autopsy)
Open Autopsy → Create New Case
Enter name → Next → Finish
Click Add Data Source
Select Disk Image / VM File
Choose .E01 file
Click Next → Finish
⏳ Wait for processing

🔍 VIEW RESULTS
File Views → All Files
Deleted Files
Recent Activity
📸 Screenshot: Any analysis result

🔹 RESULT
Disk image was successfully created and analyzed, and system data and files were observed.

FTK Imager helps in safe acquisition, and Autopsy helps in detailed forensic analysis.

Press Enter

🧪 STEP 3: Select Partition

Press Enter

🧪 STEP 4: Choose File System
Select:
👉 Other (FAT/NTFS)
Press Enter

👉 Free (only deleted files, faster)

Exp11:
AIM

To demonstrate ARP Poisoning attack in a controlled lab using Kali Linux and Ettercap.

THEORY
ARP stands for Address Resolution Protocol.
It converts IP address into MAC address in local network.
ARP has no authentication mechanism.
Attacker sends fake ARP replies to victim and gateway.
Victim traffic passes through attacker system (MITM).
Used for learning network security in authorized lab only.
PROCEDURE
Open Kali Linux.
Run Ettercap using command:
sudo ettercap -G
Select Sniff → Unified Sniffing.
Choose interface eth0.
Click Hosts → Scan for hosts.
Open Hosts List.
Select victim IP → Add to Target 1.
Select router IP → Add to Target 2.
Go to Mitm → ARP Poisoning.
Tick Sniff remote connections and start.
Observe changed ARP entries / traffic flow.
Stop attack after demonstration.
RESULT

ARP Poisoning was successfully demonstrated. Victim ARP table was modified and traffic redirection was observed.

CONCLUSION

ARP Poisoning exploits insecure ARP protocol. It can cause Man-in-the-Middle attacks, so network protection is necessary.
