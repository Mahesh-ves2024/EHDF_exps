To demonstrate common web attack scenarios using DVWA in a controlled lab environment.
THEORY
DVWA is a purposely vulnerable website for learning security.
It contains common web vulnerabilities.
Used to practice attacks safely in lab.
Helps understand both attack methods and prevention.
Security levels: Low, Medium, High.
REQUIREMENTS
Kali Linux / XAMPP / Localhost
DVWA installed
Browser
Apache + MySQL running
DETAILED PROCEDURE
STEP 1: Start DVWA
On Kali Linux
Open terminal:
Bash
sudo service apache2 start
sudo service mysql start
Open Browser
Go to:
Plain text
http://localhost/dvwa
STEP 2: Login
Default credentials:
Plain text
Username: admin
Password: password
Login successfully.
STEP 3: Reset Database
Click DVWA Security
Set level = Low
Click Submit
Then go:
Setup / Reset DB if needed.
ATTACK SCENARIO 1: SQL Injection
Steps
Open SQL Injection module
In User ID box type:
Plain text
1' OR '1'='1
Click Submit
Result
All users data may display.
Learn
Input validation missing.
ATTACK SCENARIO 2: Reflected XSS
Steps
Open XSS (Reflected)
Enter:
HTML

Submit
Result
Popup appears.
Learn

ATTACK SCENARIO 3: Command Injection
Steps
Open Command Injection
Enter:
Plain text
127.0.0.1 && whoami
Submit
Result
Ping output + username shown.
Learn

ATTACK SCENARIO 4: Brute Force
Steps
Open Brute Force module
Try common passwords:
Plain text

ATTACK SCENARIO 5: File Upload
Steps
Open File Upload

STEP 1: Install Required Packages
Open terminal:
Bash
sudo apt update
sudo apt install apache2 mariadb-server php php-mysqli php-gd libapache2-mod-php git -y

Bash
sudo service apache2 start
sudo service mariadb start
STEP 3: Check DVWA Config File
Run:
Bash
cd /var/www/html/dvwa/config
ls
If only config.inc.php.dist exists, run:
Bash
sudo cp config.inc.php.dist config.inc.php
STEP 4: Fix Permissions
Bash
sudo chmod -R 777 /var/www/html/dvwa
sudo chown -R www-data:www-data /var/www/html/dvwa
STEP 5: Enable PHP Modules
Bash
sudo phpenmod mysqli
sudo systemctl restart apache2
STEP 6: Create Database
Login MariaDB:
Bash
sudo mysql
Then run inside MySQL:
SQL
create database dvwa;
create user 'dvwa'@'localhost' identified by 'p@ssw0rd';
grant all privileges on dvwa.* to 'dvwa'@'localhost';
flush privileges;
exit;
STEP 7: Edit Config File
Open:
Bash
sudo nano /var/www/html/dvwa/config/config.inc.php
Find DB lines and set:
PHP
$_DVWA[ 'db_user' ] = 'dvwa';
$_DVWA[ 'db_password' ] = 'p@ssw0rd';
$_DVWA[ 'db_database' ] = 'dvwa';
Save:
Ctrl + O

Ctrl + X

sudo service apache2 restart
