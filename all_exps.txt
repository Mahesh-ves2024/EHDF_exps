🔥 1. FTK IMAGER (Disk + Memory)
🔹 Key Points
FTK Imager = acquisition tool
Used to create disk image (.E01) and memory dump (.mem)
Works in read-only mode
🔹 Important Lines
Disk image = exact copy of storage
Cannot save image on same drive
E01 = compressed + secure format
RAM = volatile (lost after shutdown)
🔹 Viva Q
Why imaging? → Preserve evidence
Why RAM first? → Volatile data
🔥 2. AUTOPSY (Analysis + Deleted Files)
🔹 Key Points
Autopsy = analysis tool
Used for:
Deleted files
User activity
File system analysis
🔹 Important Lines
Deleted files remain until overwritten
Uses ingest modules for analysis
🔹 Viva Q
What are deleted files? → Not erased, just hidden
What is ingest? → Automated analysis process
🔥 3. FILE RECOVERY (Recoverit / Recuva)
🔹 Key Points
Recuva / Recoverit = recovery tools
Recover deleted/lost files
🔹 Important Lines
Recovery depends on overwrite
Always save recovered file to different location
🔹 Viva Q
Why file recoverable? → Data still exists
When not recoverable? → Overwritten
🔥 4. PHOTOREC
🔹 Key Points
PhotoRec
Works using file signatures
Ignores file system
🔹 Important Lines
Works even if file system damaged
File names may change
🔹 Viva Q
Advantage? → Deep recovery
Disadvantage? → No original file names
🔥 5. REDLINE (Memory Forensics)
🔹 Key Points
Redline
Used for:
Processes
Network activity
Malware detection
🔹 Important Lines
Uses Collector
Output = AnalysisSession folder
MRI = Malware Risk Index
🔹 Viva Q
What is MRI? → Risk score
Why collector? → Gather system data
⚡ MOST IMPORTANT 1-LINERS (REMEMBER THESE)
Disk image = exact copy of disk
RAM = volatile memory
Deleted ≠ erased
Overwrite = permanent loss
FTK = acquisition
Autopsy = analysis
PhotoRec = signature-based recovery
Redline = memory + malware analysis
⚡ COMMON MISTAKES (EXAM)
Saving image on same drive ❌
Selecting wrong folder in Redline ❌
Not running collector ❌
Recovering to same location ❌
⚡ FINAL SUPER SHORT FLOW

👉 FTK → Create Image / RAM → Autopsy → Analyze → Recovery tools → Redline → Analyze processes
